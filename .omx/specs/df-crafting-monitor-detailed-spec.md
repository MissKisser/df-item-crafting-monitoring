## 2. 背景与问题

《三角洲行动》官方微信小程序已经提供物品制造状态查看能力，但用户需要进入小程序并点进对应页面，才能看到技术中心、工作台、制药台和防具台的制造情况。

本项目希望把这个「主动打开查看」的流程改成「被动实时感知」：

- 用户授权后，APP 自动读取自己的制造状态。
- APP 在 Android 通知栏中展示四类制造设施的当前状态。
- 单个物品制造完成后，APP 主动发送完成通知。
- 用户可以通过桌面卡片快速查看当前制造情况，不必进入小程序。

核心目标不是改变游戏行为，也不是提供自动化操作，而是减少查看制造状态的摩擦。

## 3. 产品定位

### 3.1 一句话定位

一个面向《三角洲行动》玩家的 Android 本地制造监控工具，用于只读监控制造状态，并通过通知栏和桌面卡片提醒用户。

### 3.2 核心价值

- 减少反复打开官方小程序的操作成本。
- 提前感知物品完成时间，避免错过制造节奏。
- 在耗电可控的前提下提供足够及时的提醒。
- 让用户掌握自己的账号数据，不把敏感登录态交给第三方服务器。

### 3.3 目标用户

| 用户类型 | 需求 | V1 支持方式 |
| --- | --- | --- |
| 普通玩家 | 不想频繁打开小程序查看制造进度 | 通知栏总览、完成提醒、桌面卡片 |
| 高频制造玩家 | 希望更及时知道物品完成 | 常驻前台通知、靠近完成时间的自适应刷新 |
| 隐私敏感用户 | 不希望账号登录态上传到第三方服务器 | 本地优先，服务器不代管登录态 |
| 高级玩家 | 愿意接受更强后台监控带来的常驻通知 | 可选前台监控模式 |

## 5. 产品原则

### 5.1 本地优先

制造数据拉取、登录态保存、完成时间计算、通知调度和桌面卡片刷新都应优先在用户设备本地完成。

服务器只能承担非敏感能力，例如：

- 版本公告；
- 非敏感配置；
- 接口适配规则；
- 崩溃诊断；
- 可选匿名遥测。

服务器不得保存：

- QQ/微信可复用登录态；
- 游戏账号登录态；
- Cookie；
- Session Token；
- 可代表用户访问制造数据的任何凭据。

### 5.2 只读访问

APP 只读取制造状态，不对游戏账号或官方服务执行任何写操作。

禁止能力：

- 启动制造；
- 领取制造结果；
- 取消制造；
- 修改物品；
- 自动操作游戏；
- 交易、出售、购买或报价。

### 5.3 失败透明

当接口失效、登录过期、网络异常或数据异常时，APP 必须把状态明确展示给用户。

禁止行为：

- 假装数据仍然实时；
- 隐藏异常；
- 后台高频重试；
- 绕过接口校验；
- 破解签名；
- 规避风控。

### 5.4 低耗电默认

默认模式优先保护电量，不追求秒级实时。用户可以明确开启更积极的监控模式，但 APP 必须说明该模式会增加常驻通知和耗电。

## 6. 核心用户流程

### 6.1 首次启动流程

1. 用户打开 APP。
2. APP 展示产品用途、只读边界和隐私说明。
3. 用户同意必要权限说明。
4. 用户选择 QQ 或微信授权。
5. APP 完成授权回调。
6. APP 尝试获取制造状态。
7. 获取成功后进入监控首页。
8. APP 引导用户开启通知权限。
9. APP 引导用户添加桌面卡片。
10. 用户选择是否开启常驻前台监控。

### 6.2 日常监控流程

1. APP 根据本地策略定期同步制造数据。
2. 同步成功后更新本地缓存。
3. 首页、通知栏和桌面卡片读取同一份本地状态。
4. 如果发现物品接近完成，APP 提高本地关注度。
5. 到达预计完成时间后，APP 发送单项完成通知。
6. 用户点击通知进入对应制造详情页。

### 6.3 登录失效流程

1. APP 同步接口返回登录失效、权限不足或无法解析的授权异常。
2. APP 停止高频后台请求。
3. 通知栏和桌面卡片显示「需要重新授权」。
4. 用户点击提示进入授权页。
5. 授权成功后恢复监控。

### 6.4 接口变更流程

1. APP 发现接口字段缺失、响应结构变化或持续解析失败。
2. APP 进入「等待适配」状态。
3. APP 降低同步频率，避免无意义重试。
4. 如果轻后端提供兼容配置，APP 拉取后尝试恢复。
5. 恢复失败时继续提示用户等待更新。

## 7. 功能规格

### 7.1 首页

首页用于展示四项制造设施的完整状态。

#### 必须展示

- 当前登录状态；
- 最近同步时间；
- 全局监控状态；
- 四项制造设施卡片：
  - 名称；
  - 当前制造物品；
  - 剩余时间；
  - 预计完成时间；
  - 状态：空闲、制造中、已完成、同步失败、未知；
- 手动刷新入口；
- 开启/暂停监控入口；
- 重新授权入口；
- 桌面卡片引导入口。

#### 状态展示规则

| 状态 | 展示要求 |
| --- | --- |
| 空闲 | 显示「空闲」或「暂无制造」 |
| 制造中 | 显示物品名、剩余时间、预计完成时间 |
| 已完成 | 显示完成状态，并提示用户进入游戏或小程序处理 |
| 同步失败 | 显示失败原因摘要和上次成功同步时间 |
| 登录失效 | 提示重新授权 |
| 数据过期 | 明确显示「数据可能已过期」 |

### 7.2 授权登录

#### V1 目标

APP 内置 QQ/微信授权入口，并在用户授权后自动获取制造数据。

#### 设计要求

- 优先使用官方 SDK 或系统可验证的授权唤起方式。
- 授权过程必须由用户主动触发。
- 授权成功后只保存最小必要登录态。
- 登录态只保存在本地安全存储中。
- 授权失败必须提供明确原因和重试入口。

#### 失败状态

| 场景 | APP 行为 |
| --- | --- |
| 用户取消授权 | 返回登录页，不启动监控 |
| 授权回调失败 | 展示错误并允许重试 |
| 登录态过期 | 暂停同步，提示重新授权 |
| 接口需要额外校验 | 停止适配，不做绕过 |
| 账号状态异常 | 停止请求并提示用户自行确认账号状态 |

### 7.3 数据同步

#### 数据来源

V1 唯一数据来源是官方微信小程序接口路径。接口细节需要在后续 API 发现阶段确认。

#### 同步策略

同步策略应采用「自适应」模型：

- 用户打开 APP 时立即同步；
- 用户手动刷新时立即同步；
- 监控开启后进行保守周期同步；
- 有制造任务接近完成时提高关注度；
- 网络异常、接口异常或登录异常时退避；
- 数据长期未变化时降低频率；
- 低电量或省电模式下主动降低频率。

#### 建议默认值

| 场景 | 建议策略 |
| --- | --- |
| APP 前台打开 | 立即同步 |
| 用户手动刷新 | 立即同步，但限制过快重复点击 |
| 常规后台监控 | 约 15 分钟级别的保守同步 |
| 物品接近完成 | 在系统允许范围内提高同步关注 |
| 数据已知完成时间 | 基于本地时间计算完成提醒 |
| 网络失败 | 指数退避 |
| 登录失效 | 停止后台同步，提示重授权 |
| 低电量 | 降低同步频率 |

### 7.4 原生通知栏总览

#### 通知类型

V1 至少需要 3 类通知：

| 通知类型 | 用途 | 建议重要性 |
| --- | --- | --- |
| 常驻监控通知 | 展示当前四项制造状态和监控开关 | 低或默认 |
| 制造完成通知 | 单个物品完成后提醒用户 | 默认或高 |
| 授权/异常通知 | 登录失效、接口异常、等待适配 | 默认 |

#### 常驻监控通知内容

必须包含：

- 监控是否开启；
- 最近同步时间；
- 下一项即将完成的物品；
- 至少一个快捷操作：
  - 暂停监控；
  - 手动刷新；
  - 打开 APP。

可选包含：

- 四项设施摘要；
- 数据过期提示；
- 重新授权入口。

#### 制造完成通知内容

必须包含：

- 制造设施名称；
- 物品名称；
- 完成时间；
- 点击后进入 APP 对应详情。

如果存在多个同时完成的物品，APP 可以合并通知，但合并后仍需清晰列出完成项。

### 7.5 桌面卡片

桌面卡片是 V1 核心能力之一，用于在 Android 桌面快速查看制造状态。

#### 必须支持

- 展示四项设施状态；
- 展示下一项完成时间；
- 展示最近更新时间；
- 点击打开 APP；
- 手动刷新入口；
- 数据过期提示。

#### 建议尺寸

| 尺寸 | 用途 |
| --- | --- |
| 2 x 2 | 极简状态，展示下一项完成和最近更新时间 |
| 4 x 2 | 标准状态，展示四项设施摘要 |
| 4 x 3 | 扩展状态，展示四项设施和更多时间信息 |

#### 刷新规则

- 同步成功后刷新；
- 手动刷新后刷新；
- APP 打开后刷新；
- 系统允许时进行周期刷新；
- 默认周期应保守，约 15 到 30 分钟级别；
- 不能保证实时刷新时，必须展示最近更新时间。

### 7.6 监控模式

V1 建议提供 2 种模式。

#### 低耗电模式（默认）

适合大多数用户。

- 不追求秒级提醒；
- 依赖系统允许的后台任务和本地预测；
- 通知延迟目标为 0 到 15 分钟；
- 更少后台唤醒；
- 更低耗电。

#### 稳定监控模式（可选）

适合愿意接受常驻通知的用户。

- 启用常驻前台通知；
- 更积极地维护监控状态；
- 靠近预计完成时间时提高关注；
- 完成提醒目标为 0 到 5 分钟；
- 明确告知用户可能增加耗电。

### 7.7 设置页

设置页至少包含：

- 登录账号状态；
- 重新授权；
- 退出登录；
- 监控总开关；
- 通知权限状态；
- 通知频道管理入口；
- 桌面卡片设置引导；
- 监控模式切换；
- 低电量策略；
- 数据清除；
- 隐私说明；
- 版本信息；
- 问题反馈。

## 9. 后台与耗电规格

### 9.1 Android 平台限制

规划和实现必须承认以下平台限制：

- WorkManager 周期任务最小间隔为 15 分钟级别，且实际执行时间可能被系统延迟。
- Doze、App Standby、低电量模式会限制后台任务、网络和闹钟。
- Android 13 及以上需要运行时通知权限。
- Android 14 及以上对前台服务类型和精确闹钟有更严格要求。
- 精确闹钟不能作为 V1 唯一可靠机制。

### 9.2 推荐后台组合

V1 推荐组合：

- WorkManager：处理保守周期同步和失败重试。
- Foreground Service：用户开启稳定监控时使用，提供可见、可停止的常驻通知。
- 本地时间预测：根据最近同步到的完成时间计算提醒窗口。
- 手动刷新：让用户随时主动拉取最新状态。
- App Widget 刷新：由同步结果和系统 Widget 机制共同驱动。

### 9.3 退避策略

| 失败类型 | 行为 |
| --- | --- |
| 普通网络错误 | 短期重试，随后指数退避 |
| 服务端 5xx | 退避，减少请求 |
| 授权失效 | 停止后台同步，提示重新授权 |
| 响应结构变化 | 标记等待适配，降低频率 |
| 账号异常 | 停止请求，提示用户检查账号 |
| 连续失败 | 延长重试间隔，并更新通知状态 |

## 10. 安全与隐私规格

### 10.1 本地安全

必须做到：

- 登录态使用 Android 安全存储能力保护。
- 本地数据库不明文保存可复用敏感凭据。
- 日志中禁止打印 Token、Cookie、Session、授权码和完整响应头。
- 崩溃日志上传前必须脱敏。
- 用户可以一键清除本地数据和退出登录。

### 10.2 后端边界

轻后端允许保存：

- APP 版本；
- 公告；
- 非敏感配置；
- 接口适配规则；
- 匿名崩溃统计；
- 匿名使用统计（需用户知情）。

轻后端禁止保存：

- 用户 QQ/微信登录态；
- 游戏账号登录态；
- Cookie；
- Session Token；
- 可复用授权凭据；
- 用户完整制造数据，除非未来另行明确并重做隐私设计。

### 10.3 用户知情

首次启动和设置页必须说明：

- APP 用途是制造状态监控；
- APP 只读访问数据；
- 敏感登录态默认只保存在本机；
- 后台监控可能带来一定耗电；
- 常驻通知用于提升监控稳定性；
- 用户可随时暂停监控、退出登录、清除数据。

### 10.4 安全红线

任何规划和实现都不得包含：

- 绕过官方接口校验；
- 破解签名；
- 规避风控；
- 隐藏异常行为；
- 模拟非用户授权行为；
- 自动化游戏操作；
- 修改、领取或提交游戏数据。

## 11. 异常与降级

### 11.1 异常状态表

| 异常 | 用户可见状态 | 系统行为 |
| --- | --- | --- |
| 未授权 | 需要登录 | 不启动监控 |
| 授权过期 | 需要重新授权 | 暂停后台同步 |
| 网络不可用 | 网络异常 | 保留缓存，等待网络恢复 |
| 接口解析失败 | 等待适配 | 降低同步频率 |
| 数据过期 | 数据可能已过期 | 显示上次同步时间 |
| 通知权限关闭 | 通知不可用 | 引导开启权限 |
| 前台通知被关闭 | 稳定监控不可用 | 切换低耗电模式或提示用户 |
| 低电量模式 | 低电量保护中 | 降低刷新频率 |

### 11.2 用户提示原则

- 提示必须说明「现在发生了什么」。
- 提示必须说明「用户能做什么」。
- 不把技术错误原样暴露给普通用户。
- 高级日志可放在诊断页面，但必须脱敏。

## 12. 验收标准

### 12.1 授权与数据

- 用户可以从 APP 发起 QQ/微信授权。
- 授权成功后，APP 能读取四项制造设施状态。
- APP 不执行任何写操作。
- 退出登录后，本地敏感状态被清除。
- 登录失效后，APP 提示重新授权，而不是持续请求。

### 12.2 通知

- 用户开启通知权限后，可以看到常驻监控通知。
- 常驻通知展示最近同步时间和下一项完成信息。
- 物品完成后，APP 发送独立完成通知。
- 稳定监控模式下，已知完成时间的提醒目标为 0 到 5 分钟。
- 低耗电或系统限制场景下，V1 可接受约 15 分钟级别延迟。
- 用户可以暂停或停止监控。

### 12.3 桌面卡片

- 用户可以添加桌面卡片。
- 桌面卡片展示四项制造状态或下一项完成状态。
- 桌面卡片显示最近更新时间。
- 数据过期时，桌面卡片明确提示。
- 点击卡片可以进入 APP。

### 12.4 可靠性

- APP 进程被系统杀死后，可以在系统允许范围内恢复监控。
- 设备重启后，用户已开启的监控状态可以恢复或提示恢复。
- 网络断开后不会高频重试。
- 接口解析失败时不会崩溃。
- 通知权限关闭时不会静默失败。

### 12.5 耗电

- 默认模式不进行秒级轮询。
- 后台同步频率受系统和电量状态约束。
- 低电量模式下降低监控频率。
- 稳定监控模式必须有明确用户开关。

## 14. 技术规划建议

### 14.1 Android 技术栈建议

推荐优先考虑原生 Android：

- Kotlin；
- Jetpack Compose；
- WorkManager；
- Room；
- DataStore；
- Android Keystore；
- App Widget 或 Jetpack Glance；
- Foreground Service；
- Notification Channels。

原因：

- 通知、后台、Widget 和系统权限都是 Android 原生强相关能力。
- V1 对后台和电量控制要求高，原生实现更可控。
- 后续可更细致适配国产系统后台策略。

### 14.2 模块划分建议

| 模块 | 职责 |
| --- | --- |
| app | 应用入口、导航、依赖装配 |
| auth | QQ/微信授权、登录态管理 |
| crafting | 制造状态领域模型 |
| data-source | 小程序接口适配 |
| sync | 同步调度、退避、后台任务 |
| notification | 常驻通知、完成通知、通知渠道 |
| widget | 桌面卡片 |
| storage | 本地缓存和安全存储 |
| settings | 用户设置和隐私控制 |
| diagnostics | 脱敏日志、崩溃诊断、问题反馈 |

### 14.3 轻后端建议

V1 可以先不实现完整后端。如果需要，后端只做：

- 配置下发；
- 公告；
- 接口适配版本控制；
- 崩溃日志接收；
- 匿名统计。

后端 API 不应需要用户账号登录态。

## 16. 风险清单

| 风险 | 影响 | V1 策略 |
| --- | --- | --- |
| 小程序接口变更 | 无法同步数据 | 适配层、配置下发、等待更新 |
| 登录态过期 | 监控中断 | 重授权流程 |
| Android 后台限制 | 通知延迟 | 前台通知模式、低耗电模式、明确延迟预期 |
| 通知权限关闭 | 无法提醒 | 权限引导和状态提示 |
| 国产系统杀后台 | 监控不稳定 | 常驻通知、用户引导、保守恢复 |
| 耗电过高 | 用户卸载 | 默认低耗电、自适应同步 |
| 用户误解为官方工具 | 信任和合规风险 | 明确非官方、只读、用户自担授权边界 |
| 日志泄露敏感信息 | 账号安全风险 | 脱敏、禁止上传登录态 |
