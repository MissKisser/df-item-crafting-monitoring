# PRD：三角洲特勤处物品制造监控 APP V1

## 1. 概述

V1 要交付一个面向公众发布的 Android APP，用于监控《三角洲行动》账号下四项物品制造设施的状态：技术中心、工作台、制药台和防具台。用户完成 QQ/微信授权后，APP 通过官方微信小程序接口路径只读获取制造状态，并在本机完成状态缓存、完成时间计算、通知调度和桌面卡片刷新。

上述能力以 M0 接口与授权可行性验证通过为前提。M0 未证明存在不违反安全红线的只读授权路径前，不能进入完整产品实现。

V1 的核心不是做「游戏助手大全」，而是把官方小程序里需要主动点进去查看的制造状态，变成 Android 系统层面的被动提醒。

## 2. 目标

- 用户可以在 APP 内完成 QQ/微信授权。
- APP 可以展示四项制造设施的当前状态。
- Android 通知栏可以持续展示制造总览。
- 单项物品制造完成后，APP 可以单独提醒用户。
- 桌面卡片可以展示当前制造状态和最近更新时间。
- 敏感登录态只保存在用户设备本地。
- 默认模式尽量低耗电，稳定监控模式允许常驻前台通知。

## 3. 非目标

- 不做 iOS。
- 不以应用商店审核作为 V1 前置目标。
- 不做服务器代管登录态。
- 不做服务器代用户轮询制造数据。
- 不做游戏内悬浮窗。
- 不做交易/价格监控。
- 不做多账号。
- 不做社交分享。
- 不做自动化游戏操作。
- 不做破解、绕过校验、签名规避、隐藏异常行为或风控规避。
- 不执行任何写入游戏账号数据的操作。

## 4. 用户故事

### 4.1 首次授权

作为玩家，我希望在 APP 内完成 QQ/微信授权，以便 APP 能自动读取我的制造状态。

验收：

- 未授权用户看到明确的授权入口。
- 用户取消授权后，APP 不启动监控。
- 授权失败时，APP 展示可理解的失败原因和重试入口。
- 授权成功后，APP 自动触发首次同步。

### 4.2 查看四项制造状态

作为玩家，我希望在首页一次看到技术中心、工作台、制药台和防具台状态，以便快速判断下一步需要处理什么。

验收：

- 每项设施展示空闲、制造中、已完成、未知或异常状态。
- 制造中状态展示物品名、剩余时间和预计完成时间。
- 数据过期时展示最近同步时间和过期提示。

### 4.3 通知栏总览

作为玩家，我希望通知栏长期显示制造概览，以便不用打开 APP 也能知道大致状态。

验收：

- 监控开启时出现常驻通知。
- 常驻通知可见、可点击、可停止。
- 常驻通知展示最近同步时间和下一项完成信息。
- 用户能从通知进入 APP 或暂停监控。

### 4.4 单项完成提醒

作为玩家，我希望物品完成时收到单独通知，以便及时进入游戏或小程序处理。

验收：

- 已知完成时间的物品到期后触发完成通知。
- 稳定监控模式目标延迟为 0 到 5 分钟。
- 受 Android 后台限制时，V1 可接受约 15 分钟级别的延迟。
- 多项同时完成时，通知可以合并，但不能重复轰炸。

### 4.5 桌面卡片

作为玩家，我希望在桌面直接查看制造状态，以便不用打开 APP。

验收：

- 用户可以添加桌面卡片。
- 卡片展示四项设施摘要或下一项完成信息。
- 卡片展示最近更新时间。
- 卡片支持点击打开 APP。
- 数据过期时卡片明确提示。

## 5. 功能需求

### 5.1 Android APP

- 使用原生 Android 能力实现通知、后台任务和桌面卡片。
- 首页展示账号状态、同步状态、四项制造状态和手动刷新。
- 设置页提供授权、退出登录、监控模式、通知状态、数据清除和隐私说明。

### 5.2 授权登录

- 提供 QQ/微信授权入口。
- 优先使用官方 SDK 或官方授权机制。
- 登录态只保存在本机安全存储中。
- 退出登录时清除本地敏感状态。
- 不把可复用登录态上传到服务器。

### 5.3 数据同步

- 数据来源为官方微信小程序接口路径。
- 同步逻辑运行在用户设备本地。
- 数据同步必须通过 API 反腐层，不允许 UI、通知、Widget 或调度器直接依赖原始响应。
- 反腐层必须维护 read-only endpoint allowlist、版本化 DTO、脱敏 fixture、错误分类和等待适配 kill switch。
- 支持 APP 打开时同步、手动同步、保守周期同步和接近完成窗口的自适应关注。
- 网络或接口异常时执行退避。
- 登录失效时停止后台同步并提示重新授权。

### 5.4 通知

- 至少定义常驻监控通知、完成通知、异常/授权通知 3 类渠道。
- Android 13 及以上必须处理运行时通知权限。
- Android 14 及以上必须声明正确的前台服务类型。
- 常驻通知必须允许用户停止监控。

### 5.5 桌面卡片

- 支持至少一个标准尺寸卡片，建议优先实现 4 x 2。
- 支持展示四项设施、下一项完成和最近更新时间。
- 支持点击进入 APP。
- 支持手动刷新入口或跳转到 APP 刷新。

### 5.6 轻后端

V1 可以不立即实现完整后端。若实现，后端只允许承担：

- 版本公告；
- 非敏感配置；
- 接口适配规则；
- 脱敏崩溃诊断；
- 可选匿名统计。

后端不得保存用户可复用登录态或完整制造数据。

## 6. 非功能需求

### 6.1 安全

- 本地凭据使用 Android Keystore、EncryptedSharedPreferences、DataStore 或等价安全机制保护。
- 日志不得包含 Token、Cookie、Session、授权码或完整响应头。
- 崩溃日志必须脱敏。
- 用户可以清除本地数据。
- 后端诊断接口必须拒收敏感字段，包括 Token、Cookie、Session、授权码、完整请求/响应头、原始接口响应、用户完整制造数据和账号标识。

### 6.2 隐私

- 首次启动说明只读用途和本地优先边界。
- 设置页提供隐私说明。
- 遥测必须可解释、可关闭或至少匿名化。

### 6.3 可靠性

- 网络失败、登录失效、接口结构变化、系统杀进程、设备重启和通知权限关闭都必须有降级路径。
- 任何异常都不得导致高频后台请求。
- 稳定监控模式的 0 到 5 分钟是 best-effort 目标，不承诺系统限制下的绝对准时。

### 6.4 耗电

- 默认模式不做秒级轮询。
- 稳定监控模式必须由用户明确开启。
- 低电量或省电模式下降低频率。
- 所有后台策略都必须承认 Android 对 WorkManager、Doze、App Standby、前台服务和精确闹钟的限制。

## 7. 发布标准

V1 可以发布时必须满足：

- 四项制造状态可以稳定展示。
- 常驻通知和完成通知通过测试。
- 桌面卡片通过测试。
- 退出登录和清除数据可用。
- 登录失效、网络异常和接口变更有可见降级。
- 日志和诊断不泄露敏感信息。
- 发布说明明确非官方、只读、本地优先和已知限制。
