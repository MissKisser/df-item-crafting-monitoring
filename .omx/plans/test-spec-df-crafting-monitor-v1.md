# 测试规格：三角洲特勤处物品制造监控 APP V1

## 1. 测试目标

验证 V1 是否按 PRD 交付 Android 本地优先制造监控能力，并证明授权、同步、通知、桌面卡片、后台恢复、耗电策略和安全红线均可接受。

## 2. 测试范围

- 授权登录；
- 四项制造状态展示；
- 本地数据缓存；
- 同步调度与退避；
- 常驻通知；
- 单项完成通知；
- 桌面卡片；
- 设置与用户控制；
- 异常降级；
- 安全与隐私；
- Android 版本和国产系统兼容性。

## 3. Unit 测试

| 模块 | 测试点 | 通过标准 |
| --- | --- | --- |
| crafting domain | 制造状态映射 | idle、crafting、completed、unknown、error 正确转换 |
| time calculator | 剩余时间计算 | 不受时区和本地时间格式影响 |
| completion detector | 完成事件识别 | 同一物品只触发一次完成事件 |
| stale policy | 数据新鲜度判断 | 15 分钟、30 分钟等阈值正确 |
| backoff policy | 退避计算 | 网络、接口、授权错误对应不同退避 |
| notification model | 通知去重键 | 不重复发送同一完成通知 |
| auth state | 登录状态机 | 未登录、已登录、已失效状态转换正确 |
| privacy sanitizer | 日志脱敏 | Token、Cookie、Session、授权码被移除 |

## 4. Integration 测试

| 流程 | 测试点 | 通过标准 |
| --- | --- | --- |
| 授权到同步 | 授权成功后首次同步 | 首页展示四项状态 |
| 手动刷新 | 用户点击刷新 | 同步任务执行且 UI 更新时间 |
| 同步到通知 | 数据显示物品完成 | 完成通知被调度 |
| 同步到 Widget | 数据更新后 | 桌面卡片刷新 |
| 登录失效 | 接口返回授权异常 | 停止后台同步并提示重授权 |
| 接口结构变化 | 响应缺字段 | 进入等待适配，不崩溃 |
| 网络失败 | 无网络同步 | 执行退避并保留缓存 |
| 退出登录 | 用户退出 | 清理本地敏感状态和停止监控 |

## 4.1 M0 Go/No-Go 测试

| 测试项 | 通过标准 |
| --- | --- |
| 授权取消 | 用户取消 QQ/微信授权后，不保存半成品登录态，不启动监控 |
| 会话过期 | 模拟登录态失效后，APP 停止后台同步并提示重授权 |
| Parser fixture replay | 脱敏样例响应可稳定解析为四项制造状态 |
| Read-only allowlist | M0 全部请求都在只读 endpoint allowlist 内 |
| No-write proof | 不存在启动制造、领取制造、取消制造或修改账号数据请求 |
| No-bypass proof | 不存在破解签名、绕过校验、隐藏异常或规避风控的步骤 |
| Sensitive-field scanner | 日志、fixture、诊断载荷中不含 Token、Cookie、Session、授权码、完整请求/响应头 |
| Go/No-Go ADR | M0 结论、证据、风险和后续路线被写入 ADR |

## 5. E2E 测试

### 5.1 首次使用

1. 安装 APP。
2. 打开 APP。
3. 阅读说明。
4. 完成授权。
5. 首次同步成功。
6. 开启通知权限。
7. 开启监控。
8. 添加桌面卡片。

通过标准：用户无需理解内部实现即可完成全部步骤，首页、通知栏和桌面卡片状态一致。

### 5.2 制造完成提醒

1. 模拟或使用测试账号返回制造中状态。
2. 本地记录预计完成时间。
3. 到达完成窗口。
4. APP 发出完成通知。
5. 用户点击通知进入对应详情。

通过标准：稳定监控模式下完成提醒目标为 0 到 5 分钟；系统限制场景下延迟不超过约 15 分钟且有合理解释。

### 5.3 后台恢复

1. 开启监控。
2. 将 APP 切到后台。
3. 模拟进程被杀。
4. 等待系统允许的后台恢复窗口。
5. 打开 APP 或等待调度恢复。

通过标准：APP 不丢失用户已开启的监控意图；无法后台恢复时能在下次打开时明确说明状态。

## 6. Observability 测试

| 指标 | 目标 |
| --- | --- |
| sync_success_rate | 同步成功率可统计，且不含敏感数据 |
| sync_latency_ms | 同步耗时可统计 |
| notification_delay_ms | 完成时间到通知触发的延迟可统计 |
| auth_error_count | 授权失败次数可统计 |
| api_parse_error_count | 接口解析失败次数可统计 |
| backoff_state | 连续失败时可诊断当前退避状态 |
| battery_mode | 可诊断当前监控模式和省电状态 |

所有观测数据必须脱敏，不得包含可复用登录态、完整响应头或制造数据明细。

后端诊断接口如存在，必须以 schema 拒收以下内容：

- Token、Cookie、Session、授权码；
- 完整请求头或响应头；
- 原始接口响应；
- 用户完整制造数据；
- QQ/微信账号标识和游戏账号标识；
- 任何可代表用户访问制造数据的凭据。

## 7. 兼容性测试

至少覆盖：

- Android 8.0：通知渠道；
- Android 13：运行时通知权限；
- Android 14：前台服务类型、精确闹钟限制；
- 无 Google Play 服务设备；
- 主流国产系统后台限制；
- 系统省电模式；
- 不同桌面启动器 Widget 行为；
- 网络从 Wi-Fi 切换到蜂窝数据；
- 设备重启。

## 8. 安全测试

- 日志扫描不出现 Token、Cookie、Session、授权码。
- Fixture 和诊断载荷扫描不出现 Token、Cookie、Session、授权码、完整请求/响应头。
- 崩溃日志脱敏。
- 退出登录后本地敏感状态不可恢复。
- 后端请求不携带可复用登录态。
- 后端请求不携带用户完整制造数据或账号标识。
- 接口异常时不进入高频请求。
- 不存在写入游戏数据的代码路径。
- 不存在绕过校验、破解签名或规避风控逻辑。

## 9. 发布阻断项

以下任一项失败则不得发布：

- 授权成功后无法展示四项制造状态；
- M0 未形成 Go/No-Go ADR；
- M0 发现必须绕过校验或破解签名才能继续；
- 完成通知不可用；
- 桌面卡片不可用；
- 登录失效后仍持续后台请求；
- 日志泄露敏感登录态；
- 退出登录无法清除敏感数据；
- 接口异常导致崩溃或高频请求；
- 实现包含任何绕过、破解、规避风控或写入游戏数据的能力。
